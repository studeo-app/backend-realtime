Studeo Realtime API

Documentación de todos los eventos Socket.IO del servidor de tiempo real. Este servidor gestiona presencia de usuarios, mensajería instantánea en salas y señalización WebRTC.

🔌 Puerto 3001 (dev) 🔐 Firebase JWT requerido Socket.IO v4.7

📡 Arquitectura de medios

Socket.IO transporta presencia, moderación, chat y señalización WebRTC. El audio, el video y la compartición de pantalla no pasan por este servidor: viajan directamente entre navegadores mediante conexiones WebRTC peer-to-peer. STUN ayuda a descubrir rutas de conexión y TURN actúa como relé de medios cuando una conexión directa no es posible.

🔐 Autenticación — Requerida en CADA conexión

Todo socket debe enviar un Firebase ID Token en el handshake. El servidor verifica la firma, expiración y proyecto antes de aceptar la conexión. Si el token es inválido o falta, la conexión es rechazada. 

// Cliente — cómo conectarse correctamente
const token = await firebase.auth().currentUser.getIdToken();
const socket = io("http://localhost:3001", {
  auth: { token }  // ← campo obligatorio
});

En caso de error de autenticación el socket recibe el evento connect_error con el mensaje Authentication error: Token not provided o Authentication error: Invalid token.

Cliente → Servidor C→S

newUser Registra presencia global del usuario ▼

Registra al usuario en la lista global de conectados y notifica a todos. El servidor obtiene uid y username directamente del token (no del payload).

Payload
// Sin payload — el servidor usa socket.data.uid / socket.data.username
socket.emit("newUser");
Emite en respuesta
usersOnline Lista actualizada de UserPresence[] → broadcast a TODOS los sockets conectados.
joinRoom Unirse a una sala de estudio ▼

Une el socket a la sala especificada. Verifica existencia en Firestore y previene que el mismo usuario (mismo uid) ocupe la misma sala desde múltiples pestañas.

Payload
socket.emit("joinRoom", {
  roomId: "abc123", // requerido
  isMuted: false,       // opcional
  isVideoOff: false     // opcional
});
Campos
CampoTipo¿Req?Descripción
roomIdstring✅ID de la sala (debe existir en Firestore)
isMutedboolean❌Estado inicial del micrófono. Default: false.
isVideoOffboolean❌Estado inicial de la cámara. Default: false.
Emite en éxito
userJoined UserPresence del nuevo miembro → otros usuarios de la sala.
roomUsers Lista actualizada UserPresence[] → todos en la sala.
Errores posibles
INVALID_ROOMroomId vacío o ausente.
ROOM_NOT_FOUNDLa sala no existe en Firestore.
ALREADY_IN_ROOMEl mismo uid ya está en esa sala desde otro socket. No bloquea unirse a una sala diferente.
leaveRoom Salir de la sala actual ▼

Elimina al socket de la sala y actualiza la presencia. Si no se pasa roomId, usa la sala activa del socket. Si el socket cierra la pestaña sin emitir este evento, el servidor ejecuta la misma lógica en disconnect.

Payload
// Opcional: sin roomId usa la sala actual del socket
socket.emit("leaveRoom");
// O con roomId explícito:
socket.emit("leaveRoom", "abc123");
Emite en éxito
userLeft { socketId, roomId } → otros usuarios de la sala.
roomUsers Lista actualizada → sala (ahora sin el usuario).

💡 Si el socket no está en ninguna sala, el evento es un no-op silencioso.

roomUsersPrevisualization Consultar presencia de una sala desde el lobby ▼

Devuelve al socket solicitante la lista de usuarios conectados a una sala sin unirlo al room de Socket.IO. Se usa en la previsualización del lobby.

Payload
socket.emit("roomUsersPrevisualization", {
  roomId: "abc123",
  socketId: socket.id // informativo; el servidor responde al socket emisor
});
Emite en éxito
roomUsers UserPresence[] → únicamente al socket solicitante.
Errores posibles
INVALID_ROOMroomId vacío o ausente.
roomMemberRemoved Salir voluntariamente o expulsar a un miembro ▼

Si uid corresponde al emisor, funciona como salida voluntaria. Si corresponde a otro usuario, solo el anfitrión puede ejecutarlo. El servidor saca al socket objetivo de la sala, limpia su presencia y actualiza la lista.

Payload
socket.emit("roomMemberRemoved", {
  roomId: "abc123",
  uid: "firebase-uid-objetivo" // opcional; default: uid del emisor
});
Emite en éxito
roomMemberRemoved{ roomId, uid } → usuario objetivo y miembros de la sala.
roomUsersLista actualizada → miembros restantes.
Errores posibles
INVALID_ROOMroomId vacío o ausente.
UNAUTHORIZEDSocket sin uid autenticado.
FORBIDDENUn usuario no anfitrión intenta expulsar a otro miembro.
roomMemberMuted Silenciar el micrófono de un miembro como anfitrión ▼

Solo el anfitrión de la sala puede emitirlo. El servidor difunde la orden; el cliente cuyo uid coincide deshabilita su track local de audio, actualiza su estado AV y emite media:status.

Payload
socket.emit("roomMemberMuted", {
  roomId: "abc123",
  uid: "firebase-uid-objetivo"
});
Emite en éxito
roomMemberMuted{ roomId, uid } → todos los sockets de la sala.
Errores posibles
INVALID_ROOMroomId vacío o ausente.
UNAUTHORIZEDSocket sin uid autenticado.
FORBIDDENEl emisor no es el anfitrión.

message:send Enviar un mensaje de chat a la sala activa ▼

Envía un mensaje a la sala en la que actualmente está el socket. El broadcast ocurre inmediatamente; la persistencia en Firestore es asíncrona y no bloquea.

Payload
socket.emit("message:send", {
  text: "Hola a todos 👋"  // ← requerido, no puede ser vacío
});
Campos
CampoTipo¿Req?Descripción
textstring✅Contenido del mensaje. Texto vacío o solo espacios es rechazado.

⚠️ NO incluir roomId — el servidor lo obtiene del estado interno del socket (in-memory store).

Emite en éxito
message:new { uid, username, text, timestamp } → TODOS en la sala (incluido el emisor).
Errores posibles
UNAUTHORIZEDEl socket no tiene uid o username válido (token sin datos de perfil).
NO_ROOMEl socket no está unido a ninguna sala.
EMPTY_MESSAGEEl campo text está vacío o contiene solo espacios en blanco.
media:status Actualizar estado de audio/video/pantalla ▼

Notifica a los demás usuarios de la sala el estado actual de los periféricos AV. El servidor actualiza la presencia y reenvía el objeto completo.

Payload
socket.emit("media:status", {
  roomId: "abc123",
  isMuted: true,
  isVideoOff: false,
  isScreenSharing: false
});
Campos
CampoTipo¿Req?Descripción
roomIdstring❌Sala donde reportar el estado. Sin roomId actualiza presencia local, pero no hace broadcast.
isMutedboolean❌Micrófono silenciado. Default: mantiene valor anterior.
isVideoOffboolean❌Cámara desactivada.
isScreenSharingboolean❌Compartiendo pantalla.
Emite en éxito
media:status UserPresence completo → otros usuarios de la sala.
Errores posibles
NOT_IN_ROOMEl roomId solicitado no coincide con la sala activa del socket.
reaction:send Enviar una reacción efímera a la sala ▼

Retransmite una reacción visual a todos los sockets de la misma sala. No se persiste en Firestore.

Payload
socket.emit("reaction:send", {
  roomId: "abc123",
  emoji: "👏"
});
Validaciones

El emisor debe estar presente en la sala, el emoji debe pertenecer a la lista permitida y se aplica un cooldown por socket.

Emite en éxito
reaction:new { id, roomId, socketId, uid, username, emoji, createdAt } → todos en la sala.
Errores posibles
REACTION_NOT_IN_ROOMEl socket no pertenece al roomId indicado.
INVALID_REACTIONEmoji vacío o fuera de la lista permitida.

Si el socket excede el cooldown, la reacción se descarta silenciosamente y no se emite reaction:new.

webrtc:offer / webrtc:answer / webrtc:ice-candidate Señalización WebRTC (relay puro) ▼

El servidor actúa como relay — reenvía el payload directamente al socket destino sin interpretarlo. Se añade fromSocketId para que el receptor sepa el origen.

Payload (igual para los tres eventos)
socket.emit("webrtc:offer", {
  roomId: "abc123",
  toSocketId: "socket-id-destino",
  offer: { /* RTCSessionDescriptionInit */ }
});
// Ídem para webrtc:answer (campo answer) y webrtc:ice-candidate (campo candidate)
Emite en éxito
webrtc:offer/answer/ice-candidate { fromSocketId, roomId, offer/answer/candidate } → socket toSocketId.
Errores posibles
INVALID_WEBRTC_SIGNALroomId o toSocketId vacío/ausente.
WEBRTC_SIGNAL_FORBIDDENEmisor y receptor no pertenecen a la misma sala.
ping Verificar conexión activa ▼ 
socket.emit("ping");
socket.on("pong", () => console.log("vivo"));

Modelos de datos

UserPresence Estado de presencia de un socket ▼ 
interface UserPresence {
  socketId:       string;        // ID único del socket (ephemeral)
  uid:            string | null; // Firebase UID del usuario
  username:       string | null; // username obtenido durante autenticación
  name:           string | null; // nombre visible del usuario
  avatarUrl:      string | null; // URL del avatar
  roomId:         string | null; // null si no está en ninguna sala
  roomOwnerUid:   string | null; // owner de la sala activa, usado para validar deleteRoom
  isMuted:        boolean;
  isVideoOff:     boolean;
  isScreenSharing:boolean;
}
MessageNewPayload Mensaje de chat emitido a la sala ▼ 
interface MessageNewPayload {
  uid:       string;  // Firebase UID del emisor
  username:  string;  // Username del perfil
  text:      string;  // Contenido del mensaje
  timestamp: string;  // ISO 8601, generado en el servidor
}

Herramientas y métodos de prueba Recomendaciones para QA ▼

Herramientas de inspección:

  • Chrome DevTools → Network: Inspeccionar eventos Socket.IO, latencia WS
  • Chrome DevTools → Performance: Graficar latencia de eventos y rendering
  • webrtc-internals (chrome://webrtc-internals): Analizar negociación ICE, bitrate, pérdida de paquetes
  • Socket.IO Monitor: Dashboard en tiempo real de eventos y sockets
  • Artillery / K6: Load testing de Socket.IO (simular múltiples usuarios)

Métodos de test:

  • Multiusuario local: Chrome + Firefox + múltiples pestañas, LAN privada
  • Simulación de latencia: Network throttling en DevTools (3G, LTE, WiFi)
  • Inyección de errores: Desconectar cableado, "Disconnect Network" en DevTools
  • Análisis de logs: Servidor emite logs estructurados para cada evento
